# Find My Kid 優化驗收規格

## 1. 驗收目標

驗證本次優化是否同時提升：
- 體驗：新手是否更快完成任務
- 品質：低信心結果是否被有效引導複核
- 效率：掃描速度與剩餘時間提示是否可用
- 韌性：取消/重試/匯出失敗補救是否完整

## 2. 功能驗收清單

### 2.1 首次導覽與前置檢查
- 首次開啟顯示任務導向 Onboarding（4 步驟）
- 最後一步顯示前置檢查（參考照、資料夾、模型）
- 可跳過導覽，重新啟動後不再自動跳出

### 2.2 主流程任務檢查卡
- 左側顯示「任務前置檢查」
- 參考照/資料夾/模型狀態需同步即時變化
- 未就緒時能顯示可行下一步提示文字

### 2.3 掃描進度與回報
- 掃描過程顯示：
  - 每秒處理量（張/秒）
  - ETA（秒）
- 掃描完成摘要可顯示 warnings
- 當有失敗分類（讀檔/嵌入）時有提醒文案

### 2.4 結果解釋與信心
- 結果卡顯示來源訊號：
  - 臉部特徵
  - 保底特徵（建議複核）
  - 未標記
- 低信心結果有明確複核建議

### 2.5 匯出與恢復
- 匯出可正常完成
- 匯出失敗可重試失敗項目
- 掃描可暫停、恢復、取消，且 UI 不應卡住

## 3. 測試資料與案例

## 案例 A：標準資料夾（300-500 張）
- 3-5 張高品質參考照
- 期待：有結果、速度提示正常、可完成匯出

## 案例 B：大資料夾（2000 張以上）
- 觀察掃描穩定性（可暫停/恢復）
- 期待：不中斷、ETA 持續更新、完成後有摘要

## 案例 C：低品質參考照
- 使用模糊、背光照片
- 期待：警示與補救建議明確（補照/放寬門檻）

## 案例 D：模型未就緒
- 人為模擬模型載入異常
- 期待：前置檢查與錯誤提示可理解，不阻塞流程

## 4. 成效門檻（建議）

- 首次任務完成率較基準提升 >= 15%
- 首次有結果比例較基準提升 >= 10%
- 低信心結果人工處理平均時間下降 >= 20%
- 錯誤後可恢復成功率 >= 95%
